Triggers — 7 nodes
▸ On Device Discovered Blocks until a matching advertisement is seen.

Blocks until a peripheral advertisement matches the configured filters (name substring, manufacturer-data hex prefix, RSSI threshold). Auto-starts a scan unless disabled.

Common flow
On Device Discovered → Connect→Read Char
Attributes
Name match
Case-insensitive substring required in the advertised local name. Empty = don't filter on name.
Manufacturer (hex)
Hex prefix the device's manufacturer data must start with. Common pattern: 4C00 for Apple beacons.
Min RSSI
Drop advertisements weaker than this dBm value. Off = no threshold.
Auto scan
When on, starts a scan on entry and stops it on match. Turn off if an upstream Start Scan already configured filters.
Timeout (s)
Maximum wait. Empty = wait forever.
Store as
Optional variable that receives the matched device address.
Example use cases
Wait for a known sensor
On Device Discovered (name=SignalHub, rssi ≥ -75) → Connect (uses stored mac).

⏱ Timer Wait on a delay, interval, or exact date.

Waits on a time condition before continuing. After delay = one-shot wait. Interval = self-looping periodic tick that runs forever until stopped. Exact date = wait until an ISO-8601 timestamp.

Common flow
Timer → Read RSSI→Append CSV
Attributes
Mode
After delay: wait N seconds, then take edge 1 once. Interval: fire repeatedly on each N-second boundary. Exact date: wait until ISO-8601 timestamp.
Seconds
Delay or interval length for After / Interval modes.
Date (ISO-8601)
Used in Exact date mode (e.g. 2026-12-31T23:00:00Z). Past dates fire immediately.
Name
Optional identifier referenced by a Stop Timer node. Defaults to the node's label.
Example use cases
Forever-periodic poll
Start → Timer (interval, 5s) → Read Char. The Read fires every 5s; press Stop to end the workflow.
Bounded periodic poll
Repeat (10) → Timer (after, 5s) → Read Char. Body executes exactly 10 times spaced 5s apart.
Scheduled run
Start → Timer (exactDate, 2026-12-31T23:00:00Z) → Start Scan → …
◾ Stop Timer Cancels a running Timer by name.

Cancels a running Timer node addressed by name. The targeted timer's pending wait is disposed immediately, no more ticks fire. Other branches and timers keep running.

Common flow
Timer→If → Stop Timer
Attributes
Timer name
Must match the target Timer node's Name field. Stopping by name lets one workflow run many timers and cancel them independently.
Example use cases
Stop on condition
Timer (name=poll, interval, 5s) → Read Char → If (battery, <, 5) → true: Stop Timer (poll) → HTTP alert.
Race two timers
Parallel → branch 1: Timer (name=A, interval, 1s) → … ; branch 2: Timer (name=B, after, 30s) → Stop Timer (A). After 30s the fast poller stops.
⦿ Capture Ad Waits for the next advertisement matching the configured filter (mac, name substring, manufacturer prefix) and stashes selected fields into prefixed variables.

Waits for the next advertisement matching the configured filter (mac, name substring, manufacturer prefix) and stashes selected fields into prefixed variables. One-shot — returns as soon as a match arrives.

Common flow
Start Scan → Capture Ad → If
Attributes
Device MAC
Specific peripheral to wait for. Empty = match by name / mfg instead.
Name contains / Manufacturer prefix
Substring filter on the advertised name and prefix filter on manufacturer bytes (uppercase hex, no separators). Combined with AND.
Variable prefix
Prefix for the captured fields: <prefix>_mac, _name, _rssi, _mfg, _services, _connectable. Defaults to ad.
Auto-scan
Start a scan on entry and stop it on match. Off when an upstream Start Scan is already running.
Timeout
How long to wait. Off = wait indefinitely.
Example use cases
Adopt the first matching sensor
Capture Ad (name~ACME, prefix=ad, timeout=10s) → Connect (mac=${ad_mac}) → Read Char.
Fingerprint by mfg payload
Capture Ad (mfg=4C00, prefix=apple) → Log saw ${apple_name} · ${apple_rssi} dBm → End.

⛨ Try / Catch Recover from failures on a branch.

Wraps its protected branch (edge 0). If any downstream step on that branch fails, execution jumps to the catch branch (edge 1) instead of failing the workflow.

Common flow
Start → Try / Catch →protected→Connect→Read Char→catch→Notify Desktop
Example use cases
Reconnect on transient failure
Try/Catch → protected: Read Char; catch: Delay 1s → Connect → Read Char again.
↺ Retry Retry a branch up to N attempts.

Wraps a branch in a retry loop. If anything downstream of edge 0 errors, the executor delays then re-enters edge 0 — up to attempts total tries before falling through to edge 1 (the give-up branch).

Common flow
Start → Retry →attempt→Connect→Read Char→gave up→Notify Desktop
Attributes
Attempts
Total tries (1 = no retry; the branch runs once and any failure falls through immediately).
Backoff (s)
Delay applied before each retry. Doesn't apply before the first attempt.
Example use cases
Reconnect on flaky link
Retry (3 attempts, 1s) → protected: Connect → Read; give-up: HTTP alert.
⏸ Debounce Trailing-edge debounce for noisy streams.

Trailing-edge debounce for noisy streams. Waits for a quiet period before continuing; a fresh arrival within the window restarts the wait, so a burst of notifications collapses into a single downstream fire once the stream goes quiet.

Common flow
Wait Notification → Debounce → HTTP Request
Attributes
Quiet period (ms)
Milliseconds of silence required before the latest token proceeds. Each new arrival restarts this timer.
Example use cases
Coalesce a chatty notify stream
On Notification (repeat) → Debounce (300 ms) → Read Char → Log. Downstream runs once per burst instead of per packet.
Settle before acting
On Notification → Debounce (500 ms) → HTTP Request. The upload fires only after the device stops sending for half a second.
⏬ Throttle Leading-edge rate limit.

Leading-edge rate limit. The first token through fires downstream immediately, then further tokens are dropped for the cooldown window. The prompt-response counterpart to Debounce — caps how often a path runs without waiting for the stream to go quiet.

Common flow
Wait Notification → Throttle → HTTP Request
Attributes
Cooldown (ms)
How long the gate stays closed after a token passes. Tokens arriving during this window are dropped.
Example use cases
Rate-limit a dashboard push
On Notification (repeat) → Throttle (500 ms) → HTTP Request. Updates the server at most twice a second even under a packet flood.
Debounce vs Throttle
Throttle fires immediately then ignores the rest of the burst; Debounce waits for the burst to end then fires once. Pick Throttle when prompt response matters.
∀ Foreach Iterates an array variable, running the body edge once per element with the current item (and index) exposed as variables.

Iterates an array variable, running the body edge once per element with the current item (and index) exposed as variables. Cleaner than Repeat + manual index arithmetic.

Common flow
JSON Extract → Foreach → Write Char
Attributes
Array variable
Holds the list to iterate — a JSON array (e.g. from JSON Extract / HTTP), or a comma/newline-separated list.
Item variable
Receives the current element on each pass. Defaults to item.
Index variable
Optional — receives the current 0-based index.
Edges
Edge 0 = body (per item); edge 1 = done. Wire the body's end back to this node, the same as Repeat.
Example use cases
Replay a command list
HTTP Request (store=resp) → JSON Extract (path=cmds, out=cmds) → Foreach (array=cmds, item=cmd) → Base64 (Decode, in=cmd, out=raw) → Write Char (${raw}); body loops back to Foreach.
Iterate a CSV line
Set Variable (vals=1, 2, 3) → Foreach (array=vals, item=v, index=i) → Log row ${i} = ${v}.
Logic — 4 nodes
? If Branch on a condition.

Branches on a condition. Edge 0 = true, edge 1 = false. Compares a Source (variable, last RSSI, last hex, isConnected) against a Right-hand value using the chosen operator.

Common flow
Read Char→Transform → If →true→Notify Desktop→false→End
Attributes
Source
What the left-hand value is: variable looks up a stored variable; lastRSSI / lastReadHex / isConnected use runtime state.
Variable Name
Only shown when Source = variable. The name to look up.
Operator
=, ≠, <, ≤, >, ≥, contains, starts with. Numeric ops parse both sides as numbers; others compare as strings.
Right-hand value
Literal value (with ${var} interpolation). When both sides start with 0x they're auto-normalized for hex comparison.
Example use cases
Low-battery alert
Read Char (store as=battery) → If (variable=battery, <, 20) → true: HTTP Request.
Match a hex response
Subscribe → Wait Notification → If (lastReadHex, =, 0x AA 01) → true: continue.

✓ Assert Fail the branch if a condition is false.

Checks a condition and fails the step if it's false. Same operand/operator semantics as the If node, but doesn't branch — it terminates the branch on failure (or jumps to the nearest Try/Catch / Retry catch edge).

Common flow
Read Char→Transform → Assert → End
Attributes
Source / Operator / Value
Same as If. Compares a variable, lastRSSI, lastReadHex, or isConnected against a literal value.
Message
Optional human-readable message included in the failure entry. Defaults to the operand-vs-expected summary.
Example use cases
Guard before write
Read MTU (store as=mtu) → Assert (variable=mtu, ≥, 100) → Write Char (large payload).
Sanity check protocol
On Notification (store as=ack) → Assert (variable=ack, starts with, 0xAA01, msg=bad ack).
⎇ Switch N-way branch.

N-way branch. Compares an input against one value per line and follows the matching outgoing edge. If nothing matches, the LAST outgoing edge fires as the default.

Common flow
Read Char→Transform → Switch →case 0→Write Char→case 1→HTTP Request→default→Log
Attributes
Source
What to compare — variable, last RSSI, last read (hex), or isConnected. Same set the If node supports.
Cases
One value per line. The Nth line maps to the Nth outgoing edge. Wire one extra outgoing edge for the default fall-through.
Example use cases
Dispatch by status byte
Read Char (status byte) → Switch (source=lastReadHex, cases: 0x00 0x01 0x02) Edges in document order → OK path / busy path / error path / default path.
Per-device routing
Set Variable (name=devType, src=isConnected) → Switch (source=variable, var=devType, cases: true false)

⊕ CRC Computes a CRC checksum over a hex payload variable.

Computes a CRC checksum over a hex payload variable. Supports CRC-8, CRC-16/CCITT-FALSE, CRC-16/Modbus, and CRC-32 — the four shapes nearly every vendor BLE protocol uses for trailing checksum bytes.

Common flow
Set Variable → CRC → Write Char
Attributes
Algorithm
Pick the variant the peripheral expects. CCITT-FALSE serialises big-endian; Modbus serialises little-endian; CRC-32 is the IEEE 802.3 variant.
Input variable
Hex string to checksum. Empty = use the most recent characteristic read.
Output variable
Destination variable. Stored in the same 0x XX XX … hex format reads produce, so a downstream Write accepts it without conversion.
Example use cases
Sign an outbound frame
Set Variable (name=payload, literal=0xA1 02 03 04) → CRC (CRC-16/Modbus, in=payload, out=crc) → Write Char (${payload} ${crc}).
Verify an incoming frame
Read Char → CRC (CRC-16/CCITT-FALSE, out=expected) → Assert (variable=expected, ==, 0x00 00).
